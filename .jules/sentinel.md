
## 2024-06-25 - Fix SSRF Bypass in FastAPI Proxy Endpoint
**Vulnerability:** The `/models` proxy endpoint in FastAPI used a string-based validation `if host and host not in TRUSTED_HOSTS:` to check the hostname before making a request. This was bypassable in two ways: 1) by providing a malformed URI or non-HTTP scheme like `file:///` which resulted in an empty host string, bypassing the conditional block due to the falsy check; and 2) the string-based check was vulnerable to DNS rebinding or DoS via delayed DNS lookup compared to a resolution-based approach, but due to constraints on blocking calls in async FastAPI routes, strict string-matching on the hostname with scheme validation (`http` / `https`) is preferred.
**Learning:** Security conditionals must not short-circuit allowing risky paths (e.g. `if host` bypassing empty hosts). Furthermore, string allowlists for hosts must be precise and ensure empty strings are rejected.
**Prevention:** Ensure scheme validation (`parsed.scheme in ("http", "https")`) and avoid short-circuit evaluation logic that inadvertently allows empty inputs (`if host not in TRUSTED_HOSTS:` correctly handles empty string, whereas `if host and host not in...` allows it). Always consider DoS/TOCTOU risks when validating hosts.
