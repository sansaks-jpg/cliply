## 2025-06-22 - [Critical] Path Traversal in File Serving Route
**Vulnerability:** A path traversal vulnerability was found in the backend's `_safe_resolve` logic for media routes. It validated that the resolved file target was within the resolved `root` directory, but the `root` directory itself was constructed using a user-controlled `task_id` (`(Path(config.STORAGE_DIR) / task_id / subdir).resolve()`). This allowed attackers to craft URLs using traversal segments like `..%2F..%2Fetc` to resolve the `root` to an unintended directory like `/etc`. The file `target` would naturally fall within this malicious `root`, bypassing the original security check.
**Learning:** Checking that a path is relative to a parent directory using `pathlib.Path.relative_to` only works if the parent directory is safe. When the parent directory construction incorporates user-controlled variables, it can be manipulated out of its intended base.
**Prevention:** Always define an absolute, immutable base path (`base_storage = Path(config.STORAGE_DIR).resolve()`). Once a sub-path structure is resolved, immediately verify that the resolved directory still falls under the base path using `root.relative_to(base_storage)` *before* verifying the file within the directory.
